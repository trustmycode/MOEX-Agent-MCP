## Описание

При необходимости реализовать лёгкий backend-прокси (Python/Node), который примет HTTP-запросы от фронтенда и проксирует их в A2A-агента, инкапсулируя детали аутентификации и формата A2A.

## Критерии приемки

- Backend-прокси предоставляет простой HTTP-API для UI (один запрос фронта → один A2A-запрос к агенту).
- Проксирование корректно обрабатывает статусы ошибок и передаёт полезные сообщения фронту.
- В коде отсутствуют хардкоды секретов; используются ENV/Secret Manager.

## Определение готовности

- Веб-UI может работать с агентом, не зная деталей A2A-контракта и аутентификации.
