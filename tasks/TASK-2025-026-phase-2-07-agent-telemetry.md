## Описание

Реализовать телеметрию агента: логирование входных/выходных запросов (без секретов), ошибки MCP/LLM, а также метрики по LLM-вызовам и полному циклу A2A-запроса.

## Критерии приемки

- Логи содержат:
  - идентификатор запроса;
  - тип/роль пользователя (если есть);
  - основные события (построение плана, вызовы MCP, LLM-генерация);
  - ошибки MCP и LLM без утечки секретов.
- Метрики включают как минимум:
  - `llm_requests_total{model}`;
  - `llm_fallback_total`;
  - гистограмму латентности полного A2A-запроса.
- Искусственный сбой основной модели приводит к увеличению `llm_fallback_total`.

## Определение готовности

- В среде разработки можно наблюдать эффективность агента и поведение fallback-логики.
