## Описание

Оформить документацию по расширенному риск-анализу (advanced-метрики, VaR, stress-тесты) и Phase 2 для RAG по новостям/аналитике как roadmap после хакатона, не реализуя сложную математику в текущей версии.

## Критерии приемки

- Подготовлен документ `SPEC_risk_analytics.md`, описывающий целевые advanced-метрики (beta по бумагам и портфелям, VaR на разных горизонтах, типы stress-сценариев), требования к данным и допущения.
- Спроектированы и задокументированы (как planned) MCP-инструменты для advanced-риска: `get_risk_factors`, `get_portfolio_var`, `get_stress_test_results` с примерными JSON Schema.
- В документации описан план RAG Phase 2 по новостям/аналитике: источники данных, частота обновления, требования к хранению и варианты архитектуры индекса (отдельный/объединённый).
- В README/ARCHITECTURE зафиксирован статус этих возможностей как roadmap после хакатона, без обещания реализации в MVP.

## Определение готовности

- Документация по advanced-риску и RAG Phase 2 согласована с бизнес-ожиданиями и техническими ограничениями платформы.
- Обозначены приоритеты введения отдельных метрик/сценариев в прод-версии и зависимости от внешних данных.
- Есть понятный план инкрементальной реализации этих возможностей после закрытия Фаз 0–6.
