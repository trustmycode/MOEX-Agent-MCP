## Описание

Реализовать простой веб-дашборд поверх A2A-агента с возможным backend-прокси: поле запроса, отображение текста и таблиц из ответа агента, базовые графики по ценам и структуре индекса/портфеля, а также подготовить сценарии демо-питча.

## Критерии приемки

- (При необходимости) реализован лёгкий backend-прокси, который принимает HTTP-запросы фронта и проксирует их в A2A-агента один-к-одному.
- Реализован одностраничный UI (SPA на выбранном стеке), который позволяет задать текстовый запрос и отображает `output.text` и `output.tables[]`.
- Для сценария «Сравни SBER и GAZP за год» UI показывает текстовое резюме, таблицу с ключевыми метриками и хотя бы один график (временной ряд или сравнение топ-5 весов).
- Подготовлен основной сценарий демо (~7 минут) и 1–2 резервных кейса; описаны шаги и ожидаемые визуализации.

## Определение готовности

- Веб-UI стабильно работает поверх A2A-агента в dev-контуре, без ручных донастроек для каждого запуска.
- Демо-сценарий можно воспроизвести по README/инструкции, включая подготовку данных и запуск окружения.
- Питч визуально демонстрирует ценность решения для CFO/аналитика и легко масштабируется под другие кейсы.

## Заметки

Эта задача агрегирует подпункты 5.1–5.3 плана архитектора (Фаза 5).
